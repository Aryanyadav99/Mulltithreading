# Java Multithreading – Interview Revenge Arc

After solving some LeetCode problems and finally becoming a Knight, I thought I was unstoppable. Built some good projects, felt like the main character.

Then one interview humbled me real quick.
The interviewer asked 3 multithreading questions. I knew 2 of them, but still struggled to explain them properly because I hadn't revised them well. That day was a reality check—my basics were there, but my revision wasn't.

Because I was always jumping from one YouTube channel to another and from one set of notes to another before interview .So ,  I decided to build a single repository for myself... and for you too. :>

## What's Inside

- Thread Class vs Runnable Interface
- Synchronization (kya scene hai bhai)
- Atomic Classes (`AtomicInteger`)
- Explicit Locks (`ReentrantLock`)
- ExecutorService (`execute()` vs `submit()`)
- Producer–Consumer Problem
- Deadlock Demonstration (kya galti se ho gaya tha)
- Personal Hinglish Notes (`DOCS` folder)
